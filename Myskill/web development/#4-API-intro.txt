jenis API 2:
RESTful API vs SOAP API 

1. Representational state transfer (RESTful API )
menggunakan metode standar CRUD untuk berinteraksi 

kelebihan
- ringan
- mudah dimengerti
- mudah dibuat
- menggunakan HTTP metode 

kekurangan
- pemanggilan tidak on-point
- bergantung pada HTTP 
- tidak konsisten


    GET,POST,PUT,DELETE
WEB     ------------     REST API   ---------- server
        JSON data 







2. simple object access protocol (SOAP API)
    protokol basis XML menggunakan protokol HTTP atau HTTPS 


kelebihan
- mudah digunakan 
- standard WSDL
- distributed computing


kekurangan
- susah ketika konfigurasi
- complex susah dikembangkan
