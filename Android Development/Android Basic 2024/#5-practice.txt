// kenapa ada etBaseAmount = findViewById(R.id.etBaseAmount)


    ini adalah kabel xml yang dihubungkan ke logika kotlin,

    kenapa penulisan katanya gitu?

    - etBaseAmount (Sisi Kiri): Ini adalah nama variabel (wadah)
        yang kamu buat di bagian atas kelas MainActivity menggunakan lateinit var.

    - = (Sama Dengan): Artinya "diisi oleh" atau "dihubungkan dengan".

    - findViewById: Ini adalah fungsi pencari otomatis milik Android. 
        Tugasnya menyisir seluruh file XML kamu untuk mencari benda tertentu.

    - R.id.etBaseAmount (Sisi Kanan): Ini adalah alamat spesifiknya.

        - R = Singkatan dari Resources (folder res).
        - id: Menandakan kita mencari berdasarkan identitas unik.
        - etBaseAmount: Nama ID yang kamu ketik di file XML (atribut android:id="@+id/etBaseAmount").





// kenapa private const val TAG = "MainActivity"
          private const val INITIAL_TIP_PERCENT = 15

          ada di luar class MainActivity


          Ibarat pengumuman di luar kantor, variabel yang berisi sebuah nilai yang tidak berubah 
          sedang variabel yang ada di class MainActivity nilainya selalu berubah,
          itu alasan kenapa di taruh di luar MainActivity class

          kenapa susunan katanya adalah private const val...
          private = hanya untuk proyek ini saja
          const = nilainya tetap, konstanta
          val = tipe jenis data

          benefit:
          - kompilasi cepat, karena komputer tidak perlu menghitung ulang
          - efisiensi memori

          tvTipPercent.text = "$INITIAL_TIP_PERCENT"
          updateTipDescription(INITIAL_TIP_PERCENT)

    + seekBarTip.progress = INITIAL_TIP_PERCENT
          ketika layar pertama kali dibuka, seek bar akan langsung nunjuk angka 15 di layar

    + tvTipPercent.text = "$INITIAL_TIP_PERCENT"
         text akan menunjukan angka 15 otomatis

        knpa ada $?
        String Templates, guna memasukan variabel ke teks. 
        Jadi, daripada menulis "15", kita memanggil variabelnya 
        agar jika nanti INITIAL_TIP_PERCENT diubah di bagian atas kode, teks ini otomatis ikut berubah.

    + updateTipDescription(INITIAL_TIP_PERCENT)
        Karena angka awalnya 15, maka fungsi ini akan langsung mengecek: "Eh, kalau 15 itu masuk kategori apa ya?". 
        Karena 15 ada di rentang in 15..19, maka teks deskripsi akan langsung muncul tulisan 
        "Good" dan warnanya berubah sesuai logika yang kamu buat.
